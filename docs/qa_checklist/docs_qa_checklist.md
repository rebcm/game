# Docs QA Checklist

## Criteria for Acceptance

1. **Presence of Screenshots**: All relevant documentation should include screenshots where applicable.
2. **Functional Links**: All links within the documentation should be functional and lead to the correct destination.
3. **Reviewed Orthography**: Documentation should be free of orthographical errors.

## Checklist

- [ ] Screenshots are included where necessary.
- [ ] All links are functional.
- [ ] Orthography has been reviewed and corrected.

## Usage

1. Review the documentation against the above criteria.
2. Mark the checkboxes as completed if the criteria are met.
3. Ensure that any new documentation is added to this checklist.

## Maintenance

This checklist should be reviewed and updated regularly to ensure it remains relevant and effective.
