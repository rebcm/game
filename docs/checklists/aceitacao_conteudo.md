# Checklist de Aceitação de Conteúdo

## Objetivo
Validar a qualidade e consistência de novas entradas de documentação no projeto.

## Critérios de Aceitação

1. **Limite de Caracteres**
   - Título: máximo de 72 caracteres
   - Descrições: máximo de 150 caracteres

2. **Presença de CTAs**
   - Verificar se CTAs (Call to Action) estão presentes onde necessário
   - CTAs devem ter texto descritivo claro (e.g., "Leia mais", "Baixe aqui")

3. **Validação de Links**
   - Todos os links devem ser válidos e acessíveis
   - Links externos devem ser verificados periodicamente

4. **Formatação Consistente**
   - Utilizar markdown consistente para títulos, subtítulos e parágrafos
   - Utilizar listas quando apropriado

5. **Conteúdo Relevante**
   - Verificar se o conteúdo é relevante para o projeto
   - Conteúdo deve estar atualizado e refletir a versão atual do projeto

## Processo de Validação

1. Revisar cada novo documento contra os critérios acima
2. Documentar quaisquer problemas encontrados
3. Corrigir ou solicitar correção dos problemas identificados

## Ferramentas Úteis

- `markdownlint` para verificar formatação markdown
- Ferramentas de validação de links como `linkchecker`

## Responsabilidades

- Desenvolvedores: seguir este checklist ao criar ou modificar documentação
- Revisores: verificar se o checklist foi seguido durante a revisão
